# Security Policy

## Supported Versions

This project is currently pre-1.0 (version 0.x). Until a stable 1.0.0
release is published, only the latest commit on the `main` branch is
supported with security updates.

| Branch / Version | Supported          |
| ---------------- | ------------------ |
| `main` (latest)  | :white_check_mark: |
| anything else    | :x:                |

## Reporting a Vulnerability

If you believe you have found a security vulnerability in this project, **do not**
open a public issue or pull request.

Instead, please report it privately using GitHub’s security advisories:

1. Go to the repository on GitHub.
2. Click on **Security** → **Report a vulnerability**.
3. Follow the prompts to create a private security advisory.

When reporting, please include as much of the following information as you can:

- A clear description of the issue and its potential impact.
- Step-by-step instructions or a proof-of-concept for reproducing the issue.
- Any workarounds you have identified.
- The environment and versions affected (OS, runtime, application version/commit).

### What to expect

- **Acknowledgement:** We aim to acknowledge receipt of your report within **3 business days**.
- **Initial assessment:** We aim to complete an initial assessment and respond with our
  evaluation (including whether we consider it a security issue and its tentative
  severity) within **10 business days** of acknowledgement.
- **Further communication:** We will keep you informed of the status of your report at
  key stages (triage complete, fix in progress, fix released, or report closed).

If we confirm that the reported issue is a security vulnerability, we will:

- Work to develop and test a fix.
- Prepare a coordinated release that includes the fix.
- Provide you with an expected timeline for the fix and public disclosure.
- Credit you for the discovery if you wish, subject to maintaining a safe disclosure
  process and any legal or policy constraints.

If we determine that the reported issue is not a security vulnerability, we will
explain why and, where appropriate, may track it as a regular issue (with your
consent) if it still represents a bug or an enhancement request.

Thank you for helping to keep this project and its users secure.
